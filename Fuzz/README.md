# Fuzz
**Коллекция invoke-задач для автоматического создания и запуска вспомогательных скриптов для:**
- целевой настройки ОС
- форматирование исходного кода
- конфигурирования
- сборки
- запуска тестов
- различные иные аспекты

## Как пользоваться:
Из корня директории Fuzz выполнить:
- На Linux (Shell):
    ```bash
    . start.sh
    ```
- На Windows (PowerShell):
    ```bat
    start.ps1
    ```

# Дополнительная информация:
## ⚠️ Только Python
Базово, весь функционал Fuzz написан на Python в концепции использования [invoke задач](https://www.pyinvoke.org/).  
Крайне важно, чтобы в данном репозитории отсутствовали любые иные ЯП, чтобы поддержать простоту и чистоту запуска задач на различных платформах.  

## Процедура автоматизации
Автоматизация процесса настройки среды работы с репозиторием происходит в два шага:
1. Настройка ОС:
   - запуск `start.sp1` (или `start.sh` на Linux)
   - установка в ОС всех перечисленного в скрипте ПО, необходимого для работы с кодовой базой проекта
   - установка `python3`
   - установка модуля `python3-venv`
2. Настройка локального Python `.venv`:
    - установка всех зависимостей из `requirements.txt`

## Настройка ОС
Настойка ОС, производится через запуск стартового скрипта `start.ps1` (или `start.sh` на Linux) путём автоматической проверки наличия в ОС целевого ПО с его установкой в случае отсутствия.  
Если есть такая возможность, то рекомендуется добавлять зависимости в `requirements.txt`, чтобы они устанавливались через `pip` и имели ограниченную область видимости в рамках текущего проекта, а не масштабах всей ОС.
